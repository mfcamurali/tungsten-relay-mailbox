## Full thesis text

# forge-cleanup 3 — Agent Thesis (upgrade)

*cleanup 2 cleared the four artifacts inside the phases. cleanup 3 recognises that coherence never
lived inside a phase — it lives in the seams between them, and in the seam where the whole suite
meets the live engine. So cleanup 3 runs across the entire calibration suite, auditing coherence
at every handoff, with the live trading logic as the fixed reference the whole chain must agree
with. A phase can be individually correct and the chain still incoherent; only whole-suite scope
can see that.*

---

## 0. Thesis

**Coherence is a property of the whole spine agreeing with itself and with what actually trades —
not of any single stage. Every one of the four artifacts is a handoff failure: intel arrives
coherent at one phase and leaves scrambled to the next. So cleanup 3 does not fix phases in
isolation; it audits every point where one phase's output becomes another's input, and the final
point where calibration's output meets the live logic, checking one thing at each seam — does what
leaves this stage still mean the same thing when it arrives at the next, and does the whole chain
still agree with the function that actually trades? Where a seam breaks meaning, that is where
coherence is lost, and that is what cleanup 3 repairs.**

---

## 1. Why whole-suite scope is the point, not a size increase

The four artifacts are all *handoff* failures:

- the exit horizon hands Phase 3B an outcome definition it never validated;
- the linear optimiser hands the live engine weights tuned for a different function;
- localisation, if done in one phase, can be re-smeared by a later pooling step;
- a learned model coherent at save time is meaningless if the loader restores it into a changed
  schema.

Each phase can pass its own check and the *chain* still be incoherent, because meaning is lost in
transit, not in place. A per-phase fix is structurally blind to this. Only a pass that spans every
seam can see intel arrive coherent and leave scrambled. That is why cleanup 3 is suite-wide by
definition — not to cover more, but to look where the failure actually is.

---

## 2. The coherence audit — every handoff, one question

cleanup 3 walks the calibration suite as a chain of handoffs and, at each seam, verifies that
meaning survives the crossing:

| Seam (output → input) | Coherence question |
|---|---|
| Exit horizon → Phase 3B | Is the "win" definition the same one 3B optimises against, and was it validated first? |
| IC / localisation → weight search | Are the cells 3B searches the same cells localisation found edge in — un-re-smeared? |
| Weight search → TP/SL | Do the brackets assume the same regime conditioning the weights were fit under? |
| TP/SL → ProbDB | Does the stored pattern carry the same outcome + regime frame it was measured in? |
| ProbDB (save) → ProbDB (load) | Does the restored model mean what it meant at save — version, schema, decay intact? |
| Whole suite → **live engine** | Does the final config score under the *same* 14-layer function live trades? (fixed reference) |

At every seam the rule is identical: the intel that leaves must still mean the same thing when it
arrives. Any seam that fails is a coherence break, logged and repaired — and the live-engine seam
is the fixed reference all internal seams must ultimately agree with (E1 enforced end-to-end, now
audited inward at every internal joint).

---

## 3. The enhancements (cleanup 2, now audited suite-wide)

**E1 — Calibrate the true objective, end to end.** Every phase scores against the actual 14-layer
live scorer, cached (risk-#2 safe). The live boundary is the fixed reference; cleanup 3 now
enforces it at *every* internal seam, not only the last.

**E2 — Localise, then optimise (un-smear the edge).** Per-cell forward-information scan first;
calibrate within edge cells, regime by regime — and the audit confirms no later phase re-pools
them.

**E3 — Fix the outcome before optimising against it.** Validate the exit horizon OOS first; the
audit confirms the same definition propagates unchanged downstream.

**E4 — Real overfit statistics.** PBO + deflated-Sharpe, so a genuine edge is kept and a lucky one
cut.

**E5 — Persist with decay.** Versioned, recency-weighted serialization; the save→load seam is
audited for meaning-preservation.

**E6 — Curriculum calibration, always-usable at any stopping point.** Highest-information cells
first; a usable validated subset at every completion point.

**E7 — Always terminate with a verdict.** Validated live-usable config, or an honest map of where
information is thin — never a silent null or a hang.

**E8 (new) — Whole-suite coherence audit.** The scope-defining upgrade: run across the entire
calibration suite, checking every handoff (§2) with the live logic as the fixed reference.
Coherence is treated as an end-to-end invariant of the spine, not a per-stage property. This is
the completion of the through-line — coherence isn't a stage, it's the whole chain staying
aligned.

---

## 4. The completion-to-live pipeline (coherence-audited)

```
localise real-edge cells (E2)
  → validate the outcome definition (E3)          [seam audit: horizon → 3B]
  → calibrate WITHIN those cells against the TRUE scorer (E1), curriculum-ordered (E6)
                                                   [seam audit: localise → search → TP/SL]
  → judge with REAL overfit statistics (E4)        → ratchet keeps genuine edge, cuts noise
  → persist the validated model, recency-weighted (E5)   [seam audit: save → load]
  → live loads and trades exactly what was validated     [seam audit: suite → live engine]
  → run always ends with a verdict + usable subset (E6/E7)
        ↑
   E8 whole-suite coherence audit spans every seam above, live logic as fixed reference
```

Each stage removes an artifact; E8 guarantees no stage silently reintroduces one at its border.
Intel stays coherent from the moment it is sensed to the moment it trades.

---

## 5. The honest result — impressive if it's real, useful if it isn't

The optimistic, defensible case is unchanged and now stronger: the null has four artifact-shaped,
handoff-shaped reasons to exist that have nothing to do with the market. cleanup 3 clears them
*and* proves, seam by seam, that no coherence is lost in transit — so if a real edge exists (and
the sign-flipping moment says one did), it now survives intact from sensing to trading. If it
emerges it will be validated, live-usable, and worth being impressed by, because it survived
scrutiny and stayed coherent end-to-end.

The honest floor holds: cleanup 3 cannot promise a number. If the feature set truly holds no
exploitable information, the impressive-and-honest output is a validated small edge plus an exact
map of where information is thin — and, now, a map of any seam where coherence was being lost,
which is itself a precise, actionable finding.
